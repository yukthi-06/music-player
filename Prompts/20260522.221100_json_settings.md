# Implementation Plan - JSON Storage for Scanning Paths

This document outlines the detailed steps to transition scanning folder paths configuration storage from Android SharedPreferences to a persistent, human-readable JSON file stored at `/sdcard/Vypeensoft/Music_Player/settings/folders.json`.

## 1. Create SettingsManager Class (SettingsManager.java)
- Establish a dedicated `SettingsManager` utility.
- Add `loadScanFolders()`:
  - Checks if `/sdcard/Vypeensoft/Music_Player/settings/folders.json` exists.
  - Reads the file's text contents, parses the JSON Array of path strings using `org.json.JSONArray`, and joins them into a newline-separated String for presentation in the UI.
- Add `saveScanFolders(String foldersStr)`:
  - Takes the newline-separated path string.
  - Splits it and builds an `org.json.JSONArray`.
  - Ensures the parent folders are fully created, then writes the pretty-printed JSON representation into the designated storage file.

## 2. Refactor Settings Activity (SettingsActivity.java)
- Remove all `SharedPreferences` fields, initializations, and editor commits.
- Wire the EditText and the button listeners (`btnSaveSettings` and `btnScanFolders`) to pull from and commit to `SettingsManager.loadScanFolders()` and `SettingsManager.saveScanFolders(paths)` respectively.

## 3. Refactor Scanning Engine (MediaScanner.java)
- Replace SharedPreferences lookups in the background scanning thread with `SettingsManager.loadScanFolders()`.
