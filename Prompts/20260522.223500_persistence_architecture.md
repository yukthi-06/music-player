# Architecture Explanation - Playback State Persistence & I/O Optimization

This document clarifies the design architecture, storage location, and resource-conscious optimization strategies used for saving the playback state.

## 1. Storage Location
- The active indices, track path, and last playback position in milliseconds are stored in the app's sandboxed private **SharedPreferences** storage under the XML preference domain `"MusicPlayerPrefs"`.
- Under the hood, Android maintains this file on secure internal flash memory at:
  `/data/data/com.vypeensoft.musicplayer/shared_prefs/MusicPlayerPrefs.xml`
- This ensures maximum performance, security, and keeps general app state protected from arbitrary external processes.

## 2. Optimized Write Cycle (No Continuous Disk I/O)
- **Continuous I/O is Avoided**: Writing to flash memory on every second's seek ticker would degrade device flash storage chips, consume high CPU resources, and severely deplete battery longevity.
- **On-Demand Write Strategy**: The app manages progress tracking entirely **in-memory** using a lightweight handler thread (`progressUpdater`).
- **Lifecycle Triggers**: Playback positions and folder information are only written to the disk **upon app exit or pause** (inside the `onPause()` override).
