# User Prompt
```
In the settings page for MP3 folder.
Also give a button to perform a "Scan Folders", so that the MP3 lists are refreshed.
Also always perform a recursive search in those folders
```

# Implementation Plan - Recursive Folder Scanning & Refreshing

This document outlines the detailed steps to add a "Scan Folders" button in the settings page and implement recursive file scanning for MP3 files in specified folders, updating the app's media library dynamically.

## 1. UI Enhancements (activity_settings.xml)
- Add a new `Button` with id `btnScanFolders` below the "Save Settings" button.
- Apply consistent premium styling to the button (vibrant accent color, primary dark text, bold text style).

## 2. Activity Logic (SettingsActivity.java)
- Reference and bind `btnScanFolders` in `onCreate`.
- On clicking `btnScanFolders`:
  1. Save the currently entered folder paths to SharedPreferences (so the latest inputs are used).
  2. Show a Toast message: `"Scanning folders..."`.
  3. Invoke `MediaScanner.scanMedia` in a background thread.
  4. Upon completion (inside the callback on the UI thread), calculate the total number of tracks and folders found.
  5. Show a success Toast message: `"Scan complete! Found X tracks in Y folders."`.

## 3. Core Scanning Logic (MediaScanner.java)
- Modify `scanMedia` to support direct recursive filesystem traversal when `allowedPaths` is provided.
- **FS Recursive Scanning**:
  - For each path in `allowedPaths`:
    - Create a `File` object.
    - If the directory exists and is a directory, invoke `scanDirectoryRecursively(File, Map<String, Folder>)`.
  - Inside `scanDirectoryRecursively`:
    - Perform a depth-first search (DFS) through subdirectories.
    - For files ending with `.mp3` (case-insensitive):
      - Extract track path, parent folder path, parent folder name, and clean the filename (remove `.mp3` extension) to use as the title.
      - Add the track to the corresponding `Folder` in the folder map.
- **Fallback**:
  - If `allowedPaths` is empty, query `MediaStore` as before to scan all music files.
- **Sorting**:
  - Keep sorting the folders alphabetically.
  - Add explicit sorting of tracks inside each folder alphabetically by track title to keep the music list organized.

## 4. Main Activity Visual Enhancements (activity_main.xml)
- **Bar Reordering**: Reorder the three main playback containers to have **Playlist** at the top, **Folder** in the middle, and **Files** at the bottom.
- **Bar Labels**: Wrap each bar's title TextView inside a vertical `LinearLayout` containing a small-font category label ("PLAYLIST", "FOLDER", "FILES") styled with `@color/accent_vibrant` and `11sp` bold text to act as a sleek category tag.
