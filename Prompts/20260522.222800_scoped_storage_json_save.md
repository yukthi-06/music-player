# Implementation Plan - Scoped Storage Permission Enforcements for Custom JSON Output

This document outlines the detailed steps to unlock storage permission paths on newer Android platforms so that direct `File` operations targeting the custom path `/sdcard/Vypeensoft/Music_Player/settings/folders.json` succeed without triggering Scope Storage "Permission Denied" failures.

## 1. Declare Storage Write and Scoped Storage Bypasses (AndroidManifest.xml)
- Declare `android.permission.WRITE_EXTERNAL_STORAGE` to support platforms up to API 29.
- Declare `android.permission.MANAGE_EXTERNAL_STORAGE` (with tools override `ScopedStorage`) to support Android 11+ (API 30+) devices, unlocking full filesystem read/write capability.
- Enable `android:requestLegacyExternalStorage="true"` in the application tag to preserve compatibility.

## 2. Request All Files Access Permission at Runtime (MainActivity.java)
- In `checkPermissions()` and `onRequestPermissionsResult()`, check if the platform is running Android 11 or higher (`Build.VERSION.SDK_INT >= Build.VERSION_CODES.R`).
- If so, verify if the app has already been granted All Files Access (`Environment.isExternalStorageManager()`).
- If missing, issue an explicit Intent to load the system settings page targeting the package so the user can easily toggle and grant this storage access.
