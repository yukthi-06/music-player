# User Prompt
```
A hierarchy has to be maintained between folder and grand parent folder.
When navigating Folder, it has to be sigling folder under the grand parent.  Not just any folder in alphabetical order.
```

# Implementation Plan - Hierarchical Folder Navigation Bounds

This document outlines the detailed steps to enforce navigation containment between the active Folder and its parent directory (Grand Parent Folder) so that folder traversal is bounded by directories that share the same parent path.

## 1. Directory Sibling Resolvers (MainActivity.java)
- Implement `getGrandParentPath(int folderIdx)`:
  - Fetches the parent path of the folder at the specified index.
- Implement `getSiblingIndices(String gpPath)`:
  - Iterates through the entire folder list and returns the indices of all `Folder` objects that share the specified parent directory path.

## 2. Bounded Navigation Implementations (MainActivity.java)
- **Folder Navigation (`nextFolder` and `prevFolder`)**:
  - Instead of flat sequential indexing, resolve the sibling folder list.
  - Cycle the current folder selection strictly within this sibling subset list.
- **Track Navigation Progression (`nextFile` and `prevFile`)**:
  - When reaching the end or beginning of a folder's track list, boundary transitions to the next/prev folder are filtered through the sibling index lists, preserving the active Grand Parent folder bounds.
