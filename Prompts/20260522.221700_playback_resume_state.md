# Implementation Plan - Remember Playback & Resume State

This document outlines the detailed steps to save the full context of what was playing (playlist, grandparent folder, active subfolder, active file, and the precise millisecond seek position) on app close and automatically resume exactly from that spot upon restarting, toggled via a new General Settings page.

## 1. Create Playback Settings Layout & Activity (activity_general_settings.xml, GeneralSettingsActivity.java)
- Build a new settings page containing an explanatory subtitle and a sleek togglable `SwitchCompat` widget for "Resume Playback State".
- Wire the switch toggle state in `GeneralSettingsActivity.java` to persist to `remember_playback_state` key in `SharedPreferences` (defaulting to `true` to ensure the best default user experience).

## 2. Declare GeneralSettingsActivity (AndroidManifest.xml)
- Add `<activity android:name=".GeneralSettingsActivity" android:label="Settings" />` so it is managed correctly by the OS.

## 3. Register Navigation Clicks (MainActivity.java)
- In the navigation drawer, map `@id/nav_folders` to the folder configuring `SettingsActivity` and `@id/nav_settings` to the new `GeneralSettingsActivity` containing general app options.

## 4. Playback State Saver (`savePlaybackState`) (MainActivity.java)
- Override `onPause()` to execute state serialization:
  - If the resume option is disabled, clear all saved playback state keys.
  - If enabled, write:
    - `saved_active_bar_index` (active bar category index).
    - `saved_playlist_index`, `saved_grand_parent_index`, `saved_folder_index`, `saved_file_index`.
    - `saved_playback_position` (millisecond seek coordinate).
    - `saved_track_path` (absolute path of the track for robust path matching on reload).

## 5. State Restorer & Playback Resumer (`restorePlaybackState`, `resumePlayback`) (MainActivity.java)
- During media scanner completion after grand parent folders are compiled:
  - Verify if a state has been saved and resume is toggled on.
  - If yes, read all indices and path identifiers.
  - **Robust Track Finder Loop**: Iterate through active lists to seek out the file matching `saved_track_path`. If found, realign indexing structures to that exact target (protecting against dynamic changes in filesystem order).
  - Call `updateUI()` and `updateActiveBar(...)`.
  - Invoke `resumePlayback(track, savedPosition)` which starts playback on `playbackManager`, issues a `seekTo(...)` command to the exact saved milliseconds progress, adjusts progress handlers, and displays the proper media metadata on screen.
